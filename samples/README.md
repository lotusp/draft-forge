# 样例测试数据

clone 下来后无需自备文件，用这里的数据即可把三个页签逐个跑通。

```bash
./start.sh          # 然后打开 http://127.0.0.1:8000/
```

> **本目录不含任何客户数据。** 全部为公开数据集或虚构样例，入库前经敏感信息扫描
> （企业名称、受控水印、真实图号/料号、导出机器路径等），结果为零命中。
> 客户提供的模型与图纸一律不入库（见根目录 `.gitignore`）。

---

## `tmpl/` —— 页签「STEP + DXF → 图纸」

| 文件 | 用途 |
|---|---|
| `spacer-block.step` | 零件模型 |
| `spacer-block-template.dxf` | 图纸模板（视图几何 + 标注 + 标题栏占位符） |

**怎么跑**：两个文件都选上 → 「生成图纸」→ 右上表单填字段（日期默认今天）→「更新图纸」→「导出 PDF」。

**能看到什么**：
- 模板里的旧视图几何被模型的真实投影替换，**标注一个字没动**
- 比例由模板反推（10:1），三个视图各自独立推出、互相印证
- 填入的字段在图上是**橙色**，模板固有内容是黑/蓝 —— 一眼看出哪些是程序填的
- 尺寸校验：`8.00 / 4.00 / 3.20` 能用包围盒验证；`0.80 / 1.50 / Ø1.60` 是特征尺寸，**验不了**（工具会如实标注）

**关于这个零件**：SPACER BLOCK，**虚构**的带肩台通孔垫块，图纸上明确印有
`FICTITIOUS PART - NOT FOR MANUFACTURE`。尺寸刻意让公差不全等于默认公差表
（`3.20 ±0.02` 从严、`0.80/1.50 ±0.1` 放松），用于演示「公差是功能决定的，
几何里推不出来」这一点。

## `step/` —— 页签「STEP → 2D 工程图」

两个模型，演示这条管道的两个不同侧面。选文件 →「开始转换」。

### `nist_ftc_10_asme1_ap242-e2.stp` —— 演示公差自动标注

这是**唯一能演示公差自动标注**的样例，因为它同时满足三个条件
（协议 AP242 + 含 B-rep + 模型侧做了语义 PMI）：

| PMI 实体 | 数量 |
|---|---|
| 语义尺寸 `DIMENSIONAL_SIZE` | 19 |
| 几何公差 `GEOMETRIC_TOLERANCE` | 33 |
| ± 公差 `PLUS_MINUS_TOLERANCE` | 23 |
| 基准引用 `DATUM_REFERENCE` | 59 |

工具做的是**标注迁移**而非标注生成 —— 图上每个公差都来自模型里的 PMI 定义，
工具不猜。换成 AP203/AP214 的模型就没有公差可迁移（协议里没有存公差的位置，见下一个）。

### `nist_ctc_05_asme1_ap203.stp` —— 演示复杂大件三视图 + 隐藏线

527×559×483 的**立体件**（法兰底座 + 圆顶盖 + 中心立柱），形状与 FTC-10、
与 tab3 的 spacer-block 差异都大。含大量圆/锥/环面/样条曲面（156 面）。

它是 **AP203、不承载 PMI**，所以图上**没有公差** —— 正好和 FTC-10 互补，
演示的是「精确投影 + 隐藏线消除」这条能力：俯视图是圆盘带一圈螺栓孔、
主视图/侧视图有内部结构的隐藏虚线。约 17s 出图。

> 这也印证了协议约束：同为公开测试件，AP242 的 FTC-10 能带公差、
> AP203 的 CTC-05 不能 —— **不是模型没标，是协议格式里没有存公差的位置**。

**两者来源**：NIST **MBE PMI Validation and Conformance Testing Project**
（美国国家标准与技术研究院，US government work，公有领域）。
文件头即写明 `from the NIST MBE PMI Validation and Conformance Testing Project`。

## `ecad/` —— 页签「ECAD → 制程图纸」

| 文件 | 用途 |
|---|---|
| `pick-place.csv` | Altium 贴装坐标（顶层 31 个元件） |
| `profile.gbr` | Gerber 板框外形层 |
| `paste_top.gbr` | Gerber 锡膏层（328 个焊盘） |

**怎么跑**：三个文件都选上 →「生成制程图纸」。基准元件留空则自动取。

**能看到什么**：真实板框 + 328 个焊盘 + 31 个元件按真实坐标落位、关键器件红框、
从基准引出的坐标式标注。

**来源**：**MosaicBus** GNSS 板（https://github.com/septentrio-gnss/MosaicBus ，**MIT 许可**）。
`pick-place.csv` 第二行原为导出机器的绝对路径（含用户名），**已脱敏**，其余内容未改。

> ⚠️ 这块板是 GNSS 板、没有光器件，所以跑出来的基准/关键器件是按**演示规则**判的
> （光路/有源器件从严 ±0.02，其余 ±0.05 取自 IPC-7351）。真实项目的基准规则与
> 关键器件清单属工艺专有知识，需由客户提供 —— 详见《技术解决方案说明》第二部分。
